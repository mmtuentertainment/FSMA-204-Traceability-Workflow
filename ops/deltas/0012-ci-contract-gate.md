# Batch 12 - CI Contract Gate

## Goal

Add a GitHub Actions contract gate that runs the repository's existing verification checks on push and pull request events, then repair the dependency alignment so the gate can start from a fresh `npm ci`. This batch is infrastructure/tooling-only.

## Files Changed

- `.github/workflows/contract-gate.yml`
- `package.json`
- `package-lock.json`
- `ops/deltas/0012-ci-contract-gate.md`

## Exact Workflow Checks

The workflow uses `actions/checkout@v4` and `actions/setup-node@v4` with Node.js `22.x` and npm cache enabled.

It runs these commands in order:

- `npm ci`
- `npm run api:check`
- `npm run typecheck`
- `npm run build`

## Verification Command Results

- Original `npm ci` result before repair - FAIL. npm reported an existing dependency resolution conflict: `openapi-typescript@7.13.0` requires peer `typescript@^5.x`, while the repo declared `typescript@^6.0.3`.
- `npm view openapi-typescript@latest version` - `7.13.0`.
- `npm view openapi-typescript@latest peerDependencies` - `{ typescript: '^5.x' }`.
- `npm view typescript@5 version --json` - latest TypeScript 5 version available was `5.9.3`.
- `npm ci` after repair from a removed `node_modules` state - PASS.
- `npm run api:check` - PASS. Redocly validated `api/openapi.yaml`, and `openapi-typescript --check` reported generated types are current.
- `npm run typecheck` - PASS.
- `npm run build` - PASS. Next.js built successfully and listed the mock-recall routes as dynamic API routes.
- `git diff --check` - PASS.
- Restricted runtime/contract diff check for `api/openapi.yaml`, `lib/api/generated/openapi-types.ts`, `app`, `lib`, and `PLAN.md` - PASS; no output.
- Workflow command presence check - PASS; the workflow contains all four requested commands and does not use `--legacy-peer-deps`, `--force`, or `npm install`.
- `npm ls typescript openapi-typescript` after repair - PASS; `openapi-typescript@7.13.0` resolves with `typescript@5.9.3`.

## Repair Decision

`openapi-typescript@latest` is still `7.13.0` and still peers on `typescript@^5.x`, so there is no smaller OpenAPI tooling upgrade that preserves the repo's existing tooling shape. The lowest-risk repair is to align the existing TypeScript devDependency from `^6.0.3` to `^5.9.3`, the latest stable TypeScript 5 release available from npm, while leaving `openapi-typescript`, production dependencies, scripts, runtime code, and CI commands unchanged.

## Package Changes

- `typescript`: `^6.0.3` -> `^5.9.3` in `package.json`.
- `node_modules/typescript` lock entry: `6.0.3` -> `5.9.3` in `package-lock.json`.
- `openapi-typescript` remains `^7.13.0` / locked `7.13.0`.
- `package-lock.json` was regenerated by npm so the lockfile is accepted by plain `npm ci`; npm also auto-recorded the dev peer `core-js@3.49.0` required by the existing `@redocly/cli -> redoc@2.5.1` chain and adjusted `@opentelemetry/api` lock metadata, without adding a new root dependency.
